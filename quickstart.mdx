## Docker setup (recommended)

The fastest way to get started is using Docker. This method automatically sets up the database, runs migrations, and seeds sample data.

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/schemalabz/opencouncil.git
    cd opencouncil
    ```
  </Step>

  <Step title="Create environment file">
    ```bash
    cp .env.example .env
    ```
    
    Edit `.env` and add your API keys. At minimum, you'll need:
    
    - `NEXTAUTH_SECRET` - Generate with `openssl rand -base64 32`
    - `RESEND_API_KEY` - For email authentication (get from [resend.com](https://resend.com))
    - `ANTHROPIC_API_KEY` - For AI features (get from [anthropic.com](https://anthropic.com))
    
    <Tip>
      See the [environment setup guide](/deployment/environment-setup) for all configuration options
    </Tip>
  </Step>

  <Step title="Start the application">
    ```bash
    ./run.sh
    ```
    
    This script will:
    - Start PostgreSQL with PostGIS
    - Run database migrations
    - Seed the database with sample data
    - Launch the Next.js dev server
    
    The application will be available at `http://localhost:3000`
  </Step>

  <Step title="Access the application">
    Open your browser and navigate to:
    
    - **Frontend**: http://localhost:3000
    - **Prisma Studio** (database GUI): Run `npm run prisma:studio`
    
    <Note>
      Sample cities and meetings are automatically seeded. Look for test users in the database for authentication.
    </Note>
  </Step>
</Steps>

### Advanced Docker options

The `run.sh` script supports several options for different use cases:

<Accordion title="Run with external database">
  Use your own PostgreSQL instance instead of Docker:
  
  ```bash
  ./run.sh --skip-db
  ```
  
  Make sure `DATABASE_URL` in `.env` points to your database.
</Accordion>

<Accordion title="Production mode">
  Build and run in production mode:
  
  ```bash
  ./run.sh --prod
  ```
</Accordion>

<Accordion title="Multiple instances">
  The script automatically detects port conflicts and uses alternative ports:
  
  - App: 3000, 3001, 3002, ...
  - Database: 5432, 5433, 5434, ...
  - Prisma Studio: 5555, 5556, 5557, ...
</Accordion>

<Accordion title="Clean up Docker resources">
  Remove all containers, volumes, and networks:
  
  ```bash
  ./run.sh --clean
  ```
</Accordion>

## Manual setup

If you prefer to run without Docker:

<Steps>
  <Step title="Clone and install dependencies">
    ```bash
    git clone https://github.com/schemalabz/opencouncil.git
    cd opencouncil
    npm install
    ```
  </Step>

  <Step title="Set up PostgreSQL database">
    You have two options:
    
    **Option 1: Use Docker for database only**
    ```bash
    docker compose up db -d
    ```
    
    **Option 2: Use your own PostgreSQL instance**
    
    Make sure it has the PostGIS extension installed:
    ```sql
    CREATE EXTENSION IF NOT EXISTS postgis;
    ```
    
    Set the `DATABASE_URL` in your `.env` file:
    ```bash
    DATABASE_URL="postgresql://user:password@host:port/database"
    ```
  </Step>

  <Step title="Run database migrations">
    ```bash
    # Run migrations
    npx prisma migrate deploy
    
    # Generate Prisma client
    npx prisma generate
    ```
  </Step>

  <Step title="Seed the database (optional)">
    Load sample data for development:
    
    ```bash
    npx prisma db seed
    ```
    
    <Info>
      This downloads seed data from GitHub and populates your database with sample cities, meetings, and users.
    </Info>
  </Step>

  <Step title="Start the development server">
    ```bash
    npm run dev
    ```
    
    The application will be available at `http://localhost:3000`
  </Step>
</Steps>

## Nix setup

OpenCouncil supports a flake-based development environment for reproducible builds.

<Steps>
  <Step title="Install Nix with flakes">
    If you don't have Nix installed:
    
    ```bash
    curl --proto '=https' --tlsv1.2 -sSf -L https://install.determinate.systems/nix | sh -s -- install
    ```
  </Step>

  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/schemalabz/opencouncil.git
    cd opencouncil
    ```
  </Step>

  <Step title="Enter the Nix development shell">
    ```bash
    nix develop
    ```
    
    This provides all dependencies (Node.js, PostgreSQL, etc.) in an isolated environment.
  </Step>

  <Step title="Run the development environment">
    ```bash
    nix run .#dev
    ```
    
    This starts the full stack with process-compose, including:
    - PostgreSQL database
    - Next.js development server
    - Database migrations
    
    <Tip>
      Use `nix run .#dev -- --help` to see all available options and database modes
    </Tip>
  </Step>
</Steps>

For detailed Nix configuration options, see the [Nix deployment guide](/deployment/nix).

## Environment variables

Create a `.env` file with these required variables:

<CodeGroup>
```bash Database
DATABASE_URL="postgresql://postgres:postgres@localhost:5432/development"
DIRECT_URL="postgresql://postgres:postgres@localhost:5432/development"
```

```bash Authentication
NEXTAUTH_URL="http://localhost:3000"
NEXTAUTH_SECRET="generate-with-openssl-rand-base64-32"
RESEND_API_KEY="re_..."
```

```bash AI & Search
ANTHROPIC_API_KEY="sk-ant-..."
ELASTICSEARCH_URL="http://localhost:9200"
ELASTICSEARCH_API_KEY="your_key"
```

```bash Tasks & Storage
TASK_API_URL="http://localhost:3005"
TASK_API_KEY="your_task_api_key"
AWS_ACCESS_KEY_ID="your_key"
AWS_SECRET_ACCESS_KEY="your_secret"
AWS_ENDPOINT_URL_S3="https://your-region.digitaloceanspaces.com"
```
</CodeGroup>

<Warning>
  Never commit your `.env` file to version control. The `.env.example` file contains all variable names with placeholder values.
</Warning>

For a complete list of environment variables, see the [environment setup guide](/deployment/environment-setup).

## Troubleshooting

<AccordionGroup>
  <Accordion title="Port already in use">
    If port 3000 is already taken, the Docker script will automatically use the next available port (3001, 3002, etc.). For manual setup, change the port in `package.json`:
    
    ```json
    "dev": "next dev -p 3001"
    ```
  </Accordion>

  <Accordion title="Database connection error">
    Ensure PostgreSQL is running and has the PostGIS extension:
    
    ```bash
    # Check if PostgreSQL is running
    docker compose ps
    
    # Or for manual setup
    psql -U postgres -c "SELECT PostGIS_version();"
    ```
  </Accordion>

  <Accordion title="Prisma migration errors">
    Reset the database and re-run migrations:
    
    ```bash
    npm run prisma:migrate:reset
    ```
    
    <Warning>This will delete all data in your database</Warning>
  </Accordion>

  <Accordion title="Seed data not loading">
    The seed script automatically downloads data from GitHub. If it fails, check your internet connection or download manually:
    
    ```bash
    curl -o prisma/seed_data.json https://raw.githubusercontent.com/schemalabz/opencouncil/main/prisma/seed_data.json
    ```
  </Accordion>

  <Accordion title="Module not found errors">
    Reinstall dependencies:
    
    ```bash
    rm -rf node_modules package-lock.json
    npm install
    ```
  </Accordion>
</AccordionGroup>
